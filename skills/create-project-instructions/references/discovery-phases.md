# Discovery Phases: create-project-instructions

Use this seven-phase sequence for project instruction generation.

## Phase 1: Project Identity And Overview
Collect:
- project name
- project purpose and audience
- current phase/status
- primary goals

## Phase 2: Technical Stack And Architecture
Collect:
- languages
- frameworks/libraries
- architecture pattern
- persistence/storage
- test frameworks
- documentation approach
- key repository structure

## Phase 3: Role Definitions
Collect:
- 3-5 roles
- for each role: when to use, behaviors, outputs, responsibilities

## Phase 4: Default Session State
Collect defaults for:
- role
- phase
- output style
- tone
- interaction mode

## Phase 5: Development Principles And Practices
Collect:
- coding standards
- quality gates
- security/compliance requirements
- preferred/forbidden patterns

## Phase 6: Current Objectives And Future Plans
Collect:
- immediate objectives
- planned features/improvements
- maintenance and technical debt priorities

## Phase 7: Key Components And Documentation
Collect:
- major components/modules
- testing strategy
- documentation inventory
- practical build/test/run commands

## Extraction Rule
After each phase, normalize findings into target section structure and flag missing required fields.
