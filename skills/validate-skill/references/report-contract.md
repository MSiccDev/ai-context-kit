# Report Contract: validate-skill

## File handling
- Write report in same directory as validated `SKILL.md`.
- Use filename: `SKILL.validation.md`.
- Overwrite existing report for the same skill.

## Required report sections
- Overall Status + Compliance Score
- Phase 1: Presence, Structure, And Frontmatter
- Phase 2: Field Constraints And Naming Parity
- Phase 3: Instruction Quality And Completeness
- Phase 4: Resource References And Safety
- Phase 5: Neutrality And Portability
- Summary (Strengths/Critical/Warnings/Enhancements)
- Recommendations (Immediate + Suggested)
- Example fixes for common failures

## Severity classification
- `Critical Issues (Must Fix)` blocks acceptance.
- `Warnings (Should Fix)` are non-blocking but tracked.
- `Enhancements (Optional)` are advisory.
