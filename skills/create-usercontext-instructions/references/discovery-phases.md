# Discovery Phases: create-usercontext-instructions

Use this phased interview sequence and do not skip phases unless the user already provided equivalent detail.

## Phase 1: Personal Identity And Professional Background
Collect:
- preferred name/label
- role/title
- organization (or placeholder)
- location/timezone
- primary technical focus and ecosystem
- optional side work/content/community context

## Phase 2: Technical Skills And Expertise
Collect:
- top programming languages
- frameworks/platforms
- daily tools
- architecture/methodology preferences
- current learning areas and skill levels
- optional certifications and language proficiency

## Phase 3: Current Projects And Goals
Collect:
- active project set with stack/platform/status/challenges
- open-source involvement where relevant
- 6-12 month goals
- expected outcomes from AI collaboration

## Phase 4: Working Style And Preferences
Collect:
- file/format preferences (docs/code/diagrams)
- organization conventions
- typical workflow and problem-solving style
- quality standards and documentation style
- optional commit style preferences

## Phase 5: Communication And Collaboration Style
Collect:
- preferred tone
- explanation depth
- terminology level
- ambiguity-handling preference
- context priorities to keep stable across turns

## Phase 6: Constraints And Limitations
Collect:
- time/schedule constraints
- resource constraints
- technical and organizational constraints
- exclusions/prohibitions (what never to suggest)

## Extraction Rule
After each phase, summarize structured findings and flag missing high-impact fields before proceeding.
