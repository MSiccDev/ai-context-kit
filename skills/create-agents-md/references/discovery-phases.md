# Discovery Phases: create-agents-md

Use this seven-phase discovery sequence before generating `AGENTS.md`.

## Phase 1: Project Identity And Purpose
Collect:
- repository purpose
- instructions vs prompts distinction
- portability constraints

## Phase 2: Repository Layout And Canonical Paths
Collect:
- canonical directories
- normative vs sample paths
- legacy structure handling

## Phase 3: Source Hierarchy And References
Collect:
- source-of-truth precedence
- key references with relative links

## Phase 4: Session-State Expectations
Collect:
- required session-state elements
- default values
- transition/no-silent-switching expectations

## Phase 5: Command Namespace And Control Surface
Collect:
- namespace policy
- command set
- alias policy

## Phase 6: Standards, Tooling, Validation Hooks
Collect:
- formatting rules
- path/link stability rules
- minimal tooling/validation touchpoints

## Phase 7: Drift-Control Workflow
Collect:
- coordinated update requirements when spec changes
- dependent artifacts requiring synchronization
