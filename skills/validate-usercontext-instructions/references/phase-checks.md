# Phase Checks: validate-usercontext-instructions

Execute these phases in order.

## Phase 1: YAML Frontmatter Validation
Check:
- valid YAML frontmatter
- required attributes: `applyTo`, `description`
- no unsupported attributes
- description format is contextual and meaningful

## Phase 2: Required Sections Validation
Check:
- all 15 required sections exist
- heading levels are correct
- section order is logical and stable
- sections contain substantive content (not placeholders)

## Phase 3: Content Completeness Validation
Check completeness for:
- professional background
- technical expertise
- current projects
- professional goals
- working style
- communication style
- constraints

## Phase 4: Format And Quality Validation
Check:
- markdown quality and consistency
- privacy/placeholder consistency
- JSON metadata validity when present

## Phase 5: Spec Compliance Validation
Check:
- instruction-based architecture (WHO/WHAT/HOW)
- portability and provider neutrality
- completeness for context-aware collaboration
